Cerebro Web UI

The Cerebro Web UI provides a read only user interface to:

  • View your account information
  • Find and understand datasets
  • Understand your access to datasets and columns
  • Inspect group permissions on any dataset for which you are an admin and have all privileges

Accessing the Web UI

To access the Web UI:

  1. Set up a Cerebro cluster.

  2. Determine the created endpoints.

    For example, run: cerebro_cli clusters endpoints <cluster_id>.

  3. Find the host and port of the component named cerebro_web:webui.

  4. In a web browser, access http://<host>:<port> (or https://<host>:<port> if using SSL).

Supported Browsers

The Cerebro UI is supported on the latest versions of the following browsers:

  • Chrome
  • Firefox
  • Microsoft Edge
  • Safari

Logging in

Use the login screen to access the Web UI. The login screen is configured to provide login options, depending upon your cluster configuration. The login options include:

  • Token login - Log in with Cerebro, JWT, and SSO tokens (if Cerebro is configured to use them).
  • Username/password login - Log in with username and password if the cluster is configured to
  • authenticate with LDAP.
  • OAuth login - Log in with OAuth if the cluster is configured for it and there is an auth server set up to handle it.

Using the Web UI

Account Details

After logging in, you land on the Home Page that shows your account information:

  • Your Cerebro username
  • The groups you belong to
  • The roles you have been granted, and the groups granting you those roles
  • Your access token, including expiration information

Tip: you can use CTRL+F or Cmd+F(Mac) to quickly find a group or role name

About Dialog

The About dialog is located at the top right corner next to your username. Click About to display:

  • UI Version
  • Build Hash
  • Planner Version
  • Planner Host/Port
  • REST API Endpoint
  • Authentication Methods Available

Copying your Access token

Home Page

Quick Access Menu option

Click your username at the top right and click Copy Access Token.

Datasets Page

Click the DATASETS navigation tab at the top to access the datasets page.

Use the Datasets page to browse, search, filter, and preview your accessible Cerebro datasets. With admin dataset access, enable group access to datasets, and set field user group read access.

Any dataset where you have any level of access appears in this list.

:heavy_exclamation_mark: If you have no datasets field read access, there could be datasets in the catalog that are not shown.

Searching and Filtering Datasets

At the top of the page you can select datasets search and filter options:

  • Search box - Search by dataset name. Any dataset name containing your input as a substring is displayed. Tip: You can quickly clear this box with the ESC key.
  • ‘Filter by database’ multi-select box - Filter the list to include only datasets in a particular database or set of databases.

Dataset Details

Click a dataset card in the datasets list to display a datasets details page on the right side.

This page contains:

  • The dataset’s metadata, including:
    • Database name
    • Dataset name
    • Owner
    • Location
    • Description
    • Created - Date and time the dataset was created
    • Metadata Changed - Date and time the metadata was last changed
    • View Definition (if dataset is an external view)
  • The dataset schema described in full:
    • Total number of columns in the schema (next to ‘Schema’ tab)
    • Each column in the dataset is shown, including:
    • Name
    • Type
    • Access (whether you have read-access to view the cells in this column)
    • If a column’s name has a gray background, it is a partitioning column

If you do not have access to a column

If your schema looks similar to the table below, it means that you do not have access to every column in the schema.

To see which groups have access to that column

  • Click See Groups in the row. A list of groups is displayed.
  • To gain access to the column, you must be added to at least one group

Dataset preview

To preview a dataset, click Show Preview in the upper right of the details page.

  • No more than 200 rows are shown (to view more, see “Dataset usage” below)
  • Column information with no access is not visible
  • The dialog can scroll vertically and horizontally

Get Started snippets

To start using this dataset, click Get Started on the upper right of the details page.

  • Sections of different sample code are displayed (Spark, Hive, Python, R and CURL)
  • To start using this dataset click the tab of your choice and copy/paste the code into the application.

Datasets with Errors

In the UI, Datasets with errors are those that failed to load due to a problem with their metadata (e.g. unsupported datatype). Datasets with errors are listed alongside the datasets that loaded properly. Clicking a dataset with an error reveals the error message on the details page on the right.

:heavy_exclamation_mark: - There may be other datasets with issues in the Cerebro deployment. The UI only lists them here if there are metadata errors. Problems in view configuration or in the data itself is not visible in the datasets page.

Admin Features

If you have ALL access on a particular object (database or dataset) in the Cerebro UI, you have access to datasets admin features and see this icon:

Review Group Access on Datasets

Any databases or datasets with which you have all access, the Admin Tab on the datasets page is visible.

This:

  • Filters the datasets list to only datasets to which you have all access
  • Reveals the “Review Access” feature for datasets to which you have Admin Access

Click Admin to see a multi-select box under Review Access. You can select a group or set of groups to see which datasets to which those groups have access. You can also filter by database to review group access to datasets only inside that database.

:heavy_exclamation_mark:Only groups with some level of access to the datasets you are admin on will be shown in this multi-select; any group not in the available list has no access to any of the datasets that are visible.

  • A column is added to every dataset showing whether the group or groups being inspected can access any field in the dataset.
    • - Selected Group, or set of groups, has no access to this dataset
    • - Selected Group, or set of groups, has some level of access to this dataset

Note: Access is treated as a logical OR of the groups’ individual access–if any of the selected groups have access to a dataset, you will see a checkmark for that group.

Review Group Access on Dataset Schema

If you have all access to a particular dataset, you will also see an Access tab on the dataset details page. There you can inspect which groups have access to which fields on a dataset (A similar experience to on the datasets page). You see:

  • A multi-select box where you can select a group or set of groups to see which fields in that dataset to which those groups have access. Similar to the Reviewing access on the datasets page, only groups with some level of access to the dataset are shown in this multi-select.

    :heavy_exclamation_mark:Any group not shown has no access to this dataset.

  • checkmark or lock icons indicate whether or not the currently selected groups have access to the field.
    • - Selected Group or groups has no access to this field
    • - Selected Group or groups has some level of access (Select or ALL) to this field.

Use the access tab to also compare two different sets of groups’ access at the same time. Do this by selecting different sets of groups in the two different access columns.

Logging Out

To logout, click your username in the upper right part of the screen. Then click “Logout”.

:heavy_exclamation_mark:Your credentials are saved in the browser until either your token expires, or you explicitly log out. Be aware of this security issue when sharing access to your computer.

Troubleshooting

The Cerebro team really values your feedback. If you experience a problem with a particular aspect of the Web UI, or notice that features do not work as described or expected, please take a screenshot or video capture and send it to us.

If possible, click the “About” link in the upper right part of the page and take note (e.g. another screenshot) of the details in the dialog that pops up. These version numbers help to diagnose and fix issues.

A note on SSL

If Cerebro is configured to run with SSL enabled, the SSL certificate must be installed on your local computer for the Web UI to work. If, upon navigating to the Web UI, the browser warns that the site is not trusted, this is an indicator that the SSL certificate needs to be installed.

On some browsers, it is possible to bypass the SSL security warning. Please note that the Cerebro Web UI will likely continue to not work, even in this case, with the Login Screen reporting this message:

Could not contact server. Check your network connection and try again later

If you see this message, it could mean you need to install the SSL certificate.
